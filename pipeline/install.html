Auto-capture speed

Faster scrolls quicker but gives Discord less time to load each screenful; it auto-slows when needed so it won’t skip messages. After changing this, re-drag (or re-Copy) the 🔥 button — the speed is baked into the button.

🧵 Capture Thread use inside one drop thread

How to use

  1. Show your bookmarks bar (Ctrl/Cmd + Shift + B) and drag both buttons onto it.
    If dragging won't stick: click Copy, then right-click the bar → Add page → paste as the URL.
  2. Open Discord in your browser. On a channel click 🔥 Capture Channel; inside a thread click 🧵 Capture Thread.
  3. A green banner says “copied N messages.”
  4. Go to the Worklist app → Dashboard → Paste capture box → paste (Ctrl/Cmd + V) → Save capture to sheet.
  5. Scroll up in Discord to load older messages, click again, paste again — duplicates are skipped automatically.

Why copy-and-paste instead of sending directly? Discord blocks its pages from contacting outside servers, so the app (not Discord) does the saving.

Capturing a lot of history? (long auto-runs)

The 🔥 Capture Channel (auto) button reads Discord as it appears on screen, so the run only continues while that window is actually showing. Three tips make long backfills painless:

  1. It now keeps your screen awake. While a run is going the display won’t sleep and the lock screen won’t kick in — so walking away no longer stops it. (You can still lock manually.)
  2. Want to use other apps meanwhile? Pop Discord into its own window (drag the tab out), then keep a thin sliver of it visible at the screen edge while you work in other apps on top. A separate window that isn’t minimized keeps running; a hidden tab does not. If it ever pauses it’ll say “bring this window to the front.”
  3. It’s faster now — it speeds up while messages keep loading and only waits when Discord needs a moment to catch up, so it won’t skip anything.
